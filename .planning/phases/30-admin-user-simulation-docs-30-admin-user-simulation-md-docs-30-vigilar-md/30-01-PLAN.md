<task type="auto">
  <name>Task 2: Create simulation.py service with core classes</name>
  <files>bot/services/simulation.py</files>
  <action>
    Create bot/services/simulation.py with three core components:

    1. ResolvedUserContext dataclass:
       - user_id: int (the actual Telegram user ID)
       - real_role: UserRole (actual role from database)
       - simulated_role: UserRole | None (override role if simulating)
       - is_simulating: bool (True if any override is active)
       - simulated_balance: int | None (optional balance override)
       - simulated_subscription_status: str | None (optional subscription override)
       - activated_at: datetime | None (when simulation started)

       Methods:
       - effective_role() -> UserRole: returns simulated_role if simulating, else real_role
       - effective_balance(real_balance: int) -> int: returns simulated_balance if set, else real_balance

    2. SimulationStore class (in-memory storage):
       - Internal dict: admin_user_id -> ResolvedUserContext
       - TTL of 30 minutes (auto-expire old entries)
       - Methods:
         * activate_simulation(admin_id, mode, balance=None, subscription_status=None) -> ResolvedUserContext
         * deactivate_simulation(admin_id) -> bool
         * get_context(admin_id) -> ResolvedUserContext | None
         * is_simulating(admin_id) -> bool
         * cleanup_expired() -> int (number removed)

    3. SimulationService class:
       - Initialized with session and bot (like other services)
       - Has SimulationStore instance as class-level singleton (shared across instances)
       - Core method: resolve_user_context(user_id) -> ResolvedUserContext
           * Checks if user is admin (Config.is_admin)
           * If admin and has active simulation: return stored context
           * If admin and no simulation: create context with real_role, is_simulating=False
           * If not admin: return context with real_role only (no simulation possible)
       - Methods for admin control:
           * start_simulation(admin_id, mode, balance=None, subscription_status=None)
           * stop_simulation(admin_id)
           * get_simulation_status(admin_id) -> dict with mode, time_remaining, etc.

    Safety guards:
    - Simulation only works for admins (Config.is_admin check)
    - Non-admins cannot activate simulation (return error/False)
    - Context is isolated per admin (dict key is admin_id)

    Follow existing service patterns from bot/services/role_detection.py and bot/services/subscription.py:
    - Use proper type hints
    - Use Google-style docstrings
    - Use logger = logging.getLogger(__name__)
    - Import Config from config module for admin checks
  </action>
  <verify>python -c "from bot.services.simulation import SimulationService, SimulationStore, ResolvedUserContext; print('Imports OK')" 2>&1</verify>
  <done>simulation.py exists with all three classes, imports work without errors</done>
</task>
