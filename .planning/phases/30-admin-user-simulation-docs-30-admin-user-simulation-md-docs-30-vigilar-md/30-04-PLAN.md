  <action>
    Add simulation safety checks to WalletService methods:

    1. Import at top of file:
       from bot.services.simulation import SimulationStore

    2. Add helper method to check simulation:
       ```python
       def _check_simulation_block(self, user_id: int, action: str) -> tuple[bool, str]:
           """Check if user is in simulation mode and block state-changing actions.

           Returns:
               Tuple of (is_blocked, error_message)
           """
           if SimulationStore.is_simulating(user_id):
               return True, f"🎩 Acción bloqueada: No se pueden {action} durante la simulación."
           return False, ""
       ```

    3. Add checks to state-changing methods:
       - earn_besitos(): Check at start, return (False, error_msg, None) if blocked
       - spend_besitos(): Check at start, return (False, error_msg, None) if blocked
       - admin_credit(): Check at start (even for admins)
       - admin_debit(): Check at start

    4. Example integration:
       ```python
       async def earn_besitos(self, user_id, amount, transaction_type, ...):
           is_blocked, error_msg = self._check_simulation_block(user_id, "ganar besitos")
           if is_blocked:
               logger.warning(f"Blocked earn_besitos for user {user_id} during simulation")
               return False, error_msg, None
           # ... rest of method
       ```

    5. Read-only methods (get_balance, get_transaction_history) should NOT be blocked - they should return simulated values via context resolution.

    Follow existing error handling patterns in the file.
  </action>
