  <action>
    Update the cmd_start handler in bot/handlers/user/start.py to check user_context from data dict before falling back to Config.is_admin().

    Current code (around line 67):
    ```python
    # Verificar si es admin PRIMERO
    is_admin = Config.is_admin(user_id)
    ```

    The handler needs to:
    1. Accept the `data` parameter in the handler signature (it's currently not using middleware data)
    2. Check if user_context exists in data and is simulating
    3. If simulating, use the effective_role() from context instead of Config.is_admin()
    4. If not simulating or no context, fall back to Config.is_admin()

    Changes needed:

    1. Update the handler signature to accept data parameter:
       ```python
       async def cmd_start(message: Message, state: FSMContext, data: dict):
       ```
       OR if using aiogram's dependency injection:
       The handler likely needs to use middleware-provided data via a different mechanism.

    2. Replace the admin check logic (around line 67):
       From:
       ```python
       # Verificar si es admin PRIMERO
       is_admin = Config.is_admin(user_id)
       ```

       To:
       ```python
       # Verificar si es admin PRIMERO (respetando simulación si está activa)
       user_context = data.get("user_context") if 'data' in locals() else None
       if user_context and user_context.is_simulating:
           # During simulation, use effective role from context
           effective_role = user_context.effective_role()
           is_admin = False  # Admin is simulating a user role, not acting as admin
       else:
           is_admin = Config.is_admin(user_id)
       ```

    Note: The handler may need to be converted to use aiogram's middleware data injection pattern. Check how other handlers in the codebase access middleware-injected data (like container, session).

    Alternative approach if handler signature cannot be changed:
    - The handler currently gets container via `state.middleware_data.get("container")` or similar
    - Check if user_context is available in middleware_data
    - Use that to determine effective role
