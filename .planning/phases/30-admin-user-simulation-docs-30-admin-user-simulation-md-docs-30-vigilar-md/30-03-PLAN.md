  <name>Task 1: Create simulation control handlers</name>
  <files>bot/handlers/admin/simulation.py</files>
  <action>
    Create bot/handlers/admin/simulation.py with handlers:

    1. Router setup:
       - from aiogram import Router, F
       - from aiogram.types import Message, CallbackQuery
       - from aiogram.filters import Command
       - router = Router(name="admin_simulation")

    2. Command handler /simulate:
       - Decorator: @router.message(Command("simulate"))
       - Extract container from data (injected by middleware)
       - Get current simulation status via container.simulation.get_simulation_status(user_id)
       - Build status message with Lucien's voice (🎩):
         * If simulating: "⚠️ Modo de simulación activo: VIP\n🎩 Lucien: Su solicitud de simular el rol VIP ha sido procesada..."
         * If not simulating: "🎩 Lucien: Bienvenido al sistema de simulación..."
       - Show current mode and time remaining if active
       - Include mode selector inline keyboard (VIP, FREE, REAL/Desactivar)

    3. Callback handlers for mode selection:
       - simulation:set:vip -> activate VIP simulation
       - simulation:set:free -> activate FREE simulation
       - simulation:set:real -> deactivate (return to real)
       - simulation:refresh -> refresh status display

    4. Each callback handler:
       - Answer callback query immediately (avoid "loading...")
       - Call appropriate SimulationService method
       - Edit message with new status
       - Show confirmation with Lucien's voice

    5. Safety checks:
       - All handlers check Config.is_admin(user_id) before processing
       - Non-admins get "No tiene permisos" message
       - Log all simulation activations/deactivations

    6. Visual banner when simulating:
       - Add function get_simulation_banner(context) -> str
       - Returns "⚠️ MODO SIMULACIÓN: VIP | Tiempo restante: 28 min" or empty string
       - Use this in all responses when context.is_simulating

    Follow patterns from bot/handlers/admin/main.py for:
    - Handler structure
    - Lucien voice messages
    - Keyboard building
    - Error handling
