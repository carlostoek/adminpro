  <action>
    Create bot/middlewares/simulation.py with SimulationMiddleware class:

    1. Class SimulationMiddleware(BaseMiddleware):
       - Follow same pattern as RoleDetectionMiddleware
       - Extract user from event (Message or CallbackQuery)
       - Get container from data dict (injected by DatabaseMiddleware)
       - Call container.simulation.resolve_user_context(user.id)
       - Inject result into data["user_context"]
       - Log debug message with simulation status

    2. Key implementation details:
       - Must check if container exists in data (DatabaseMiddleware runs first)
       - If no container: log warning and continue (don't break handler)
       - If user is None: continue without injection
       - Use try/except around resolution to prevent breaking handlers on errors

    3. Code structure:
       ```python
       async def __call__(self, handler, event, data):
           user = extract from event
           if not user:
               return await handler(event, data)

           container = data.get("container")
           if not container:
               logger.warning("No container in data, skipping context resolution")
               return await handler(event, data)

           try:
               context = await container.simulation.resolve_user_context(user.id)
               data["user_context"] = context
               logger.debug(f"Context injected for user {user.id}: simulating={context.is_simulating}")
           except Exception as e:
               logger.error(f"Failed to resolve user context: {e}")

           return await handler(event, data)
       ```

    Follow exact patterns from bot/middlewares/role_detection.py for:
    - Imports (BaseMiddleware, TelegramObject types)
    - User extraction from Message/CallbackQuery
    - Logging setup
    - Error handling
  </action>
