## Critical Issues to Fix

### C-001: Race Condition in redeem_vip_token (lines 216-312)
**Problem:** Token can be redeemed multiple times by concurrent requests.
**Current flow:**
1. SELECT token (check not redeemed)
2. (Time window for race)
3. UPDATE token as redeemed
4. CREATE VIPSubscriber

**Fix:** Single atomic UPDATE with rowcount check

### C-002: Race Condition in create_free_request (lines 713-754)
**Problem:** User can spam multiple Free requests before first is processed.
**Current flow:**
1. SELECT check existing pending request
2. (Time window for race)
3. INSERT new request

**Fix:** Database-level UNIQUE constraint + UPSERT pattern
</context>

```python
result = await self.session.execute(
```

Replace with atomic pattern:
```python
from sqlalchemy import update

async def redeem_vip_token(self, token_str: str, user_id: int) -> Tuple[bool, str, Optional[VIPSubscriber]]:
    # Atomic UPDATE: mark token as redeemed ONLY if not already redeemed
    # Returns rowcount = 1 if successful, 0 if token was already redeemed

    result = await self.session.execute(
        update(InvitationToken)
        .where(
            InvitationToken.token == token_str,
            InvitationToken.redeemed == False,
            InvitationToken.expires_at > datetime.utcnow()
        )
        .values(
            redeemed=True,
            redeemed_by=user_id,
            redeemed_at=datetime.utcnow()
        )
    )

    if result.rowcount == 0:
        # Token was already redeemed, expired, or doesn't exist
        logger.warning(f"Token {_mask_token(token_str)} race condition or invalid")
        return (False, "Token inválido, expirado o ya canjeado", None)

    # Token successfully marked as redeemed - now create VIP subscriber
    # (This part remains similar but without the token check)

    # Get token details for subscription creation
    token_result = await self.session.execute(
        select(InvitationToken).where(InvitationToken.token == token_str)
    )
    token = token_result.scalar_one()

    # Create VIP subscriber
    expiry_date = datetime.utcnow() + timedelta(hours=token.duration_hours)
    subscriber = VIPSubscriber(
        user_id=user_id,
        token_id=token.id,
        expiry_date=expiry_date,
        status="active"
    )
    self.session.add(subscriber)
    await self.session.flush()

    logger.info(f"Token {_mask_token(token_str)} redeemed by user {_mask_user_id(user_id)}")
    return (True, "Token canjeado exitosamente", subscriber)
```

Key changes:
1. Use UPDATE with WHERE clause instead of SELECT then UPDATE
2. Check rowcount to detect if token was already redeemed
3. Only proceed with VIPSubscriber creation if UPDATE succeeded
4. Remove the separate token.redeemed = True assignment

```python
from sqlalchemy import insert

async def create_free_request(self, user_id: int) -> Tuple[bool, str, Optional[FreeChannelRequest]]:
    """
    Create a free channel request with race condition protection.

    Returns:
        Tuple of (success, message, request_object)
    """
    # Method 1: Use INSERT with ON CONFLICT DO NOTHING (PostgreSQL)
    # For SQLite: Use transaction with explicit locking

    # Start by trying to insert directly - let the database enforce uniqueness
    try:
        request = FreeChannelRequest(
            user_id=user_id,
            request_date=datetime.utcnow(),
            processed=False,
            processed_at=None
        )
        self.session.add(request)
        await self.session.flush()

        logger.info(f"Free request created for user {_mask_user_id(user_id)}")
        return (True, "Solicitud creada exitosamente", request)

    except IntegrityError:
        # Unique constraint violation - user has pending request
        await self.session.rollback()

        # Get existing request for error message
        existing = await self.get_free_request(user_id)
        if existing:
            wait_time = await self._get_wait_time_display(existing)
            return (False, f"Ya tienes una solicitud pendiente. Tiempo estimado: {wait_time}", None)

        return (False, "Ya tienes una solicitud pendiente", None)
```

Alternative for SQLite without partial unique indexes:
```python
async def create_free_request(self, user_id: int) -> Tuple[bool, str, Optional[FreeChannelRequest]]:
    # Use SELECT FOR UPDATE to lock the user's requests
    from sqlalchemy import select, func

    # Lock all requests for this user to prevent concurrent inserts
    result = await self.session.execute(
        select(FreeChannelRequest)
        .where(FreeChannelRequest.user_id == user_id)
        .with_for_update()
    )
    existing_requests = result.scalars().all()

    # Check for unprocessed request
    for req in existing_requests:
        if not req.processed:
            wait_time = await self._get_wait_time_display(req)
            return (False, f"Ya tienes una solicitud pendiente. Tiempo estimado: {wait_time}", None)

    # No pending request - safe to create
    request = FreeChannelRequest(
        user_id=user_id,
        request_date=datetime.utcnow(),
        processed=False,
        processed_at=None
    )
    self.session.add(request)
    await self.session.flush()

    logger.info(f"Free request created for user {_mask_user_id(user_id)}")
    return (True, "Solicitud creada exitosamente", request)
```
