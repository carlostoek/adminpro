The model defines a PARTIAL unique index `uq_user_pending_request` on `user_id` where `pending_request = 1`. These are different — the model provides the race condition protection (C-002) only if the partial unique index exists.

The current migration:
- Creates `idx_pending_request` on `['pending_request']` — plain, non-unique, wrong column
- Does NOT create `uq_user_pending_request` on `['user_id']` with partial condition

**Fix:** Replace the index creation at the end of the `if not column_exists:` block.

Find this line in the upgrade() function (around line 81):


Replace with dialect-aware partial index creation:
