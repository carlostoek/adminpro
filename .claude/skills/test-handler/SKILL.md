# test-handler

Generate comprehensive pytest scaffolding for aiogram handlers in the ONDA 1 Telegram bot architecture.

## When to use

- User asks to "create tests" for a handler file
- User mentions "test handler" or "handler tests"
- User needs test boilerplate for new handlers
- User wants to add tests to the test suite for handlers

## What this skill generates

1. **Complete test file** with proper imports and fixtures
2. **ServiceContainer mocks** with async patch decorators
3. **FSM state context mocks** for stateful handlers
4. **Message/CallbackQuery fixtures** using aiogram test factories
5. **Voice consistency assertions** (🎩 for admin/Lucien, 🫦 for user/Diana)
6. **Test cases** covering success paths and common error scenarios

## Output structure

The generated test file follows this pattern:

```python
"""
Tests for <handler_name> handlers.

Generated by test-handler skill.
"""

import pytest
from unittest.mock import AsyncMock, patch, MagicMock
from aiogram.types import Message, CallbackQuery, User, Chat

from bot.handlers.<module> import <handler_functions>
from bot.services.container import ServiceContainer


# Fixtures
@pytest.fixture
def mock_container():
    """Mock ServiceContainer with all services."""
    container = MagicMock(spec=ServiceContainer)
    container.subscription = AsyncMock()
    container.channel = AsyncMock()
    container.config = AsyncMock()
    return container


@pytest.fixture
def mock_message():
    """Mock Message for testing."""
    return Message(
        message_id=1,
        date=None,
        chat=Chat(id=123456, type="private"),
        from_user=User(id=123456, is_bot=False, first_name="Test"),
        text="Test message"
    )


@pytest.fixture
def mock_state():
    """Mock FSMContext for stateful handlers."""
    state = AsyncMock()
    state.get_data = AsyncMock(return_value={})
    state.set_state = AsyncMock()
    state.update_data = AsyncMock()
    return state


# Test functions follow...
```

## Usage

```
/test-handler file=bot/handlers/admin/vip.py
```

Or provide the handler file path when prompted.

## Voice assertion patterns

Always include these assertions for message tests:

```python
# For admin handlers (Lucien voice)
assert "🎩" in sent_message.text
assert any(word in sent_message.text.lower() for word in ["su", "usted", "le"])

# For user handlers (Diana voice)
assert "🫦" in sent_message.text
assert any(word in sent_message.text.lower() for word in ["tú", "eres", "estás"])
```
